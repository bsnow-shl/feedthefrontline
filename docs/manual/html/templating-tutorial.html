3.2. An example

A basic CPF template looks something like this:

<html>
 <head>
   <title>page title</title>
 </head>
 <body>
 ##PART##
 </body>
</html>

Imagine there's a file called page.html that looks like this:

Hello world!

Notice that there is no markup whatsoever in this page. It's just one line with two words on it. When a web browser pulls page.html through the CPF, this is what it will see:

<html>
 <head>
   <title>page title</title>
 </head>
 <body>
Hello world!
 </body>
</html>

The magic token ##PART## in the template has been replaced with the contents of the part.

The part can do more than appear verbatim inside the template: certain elements of the part are extracted and substituted into the template. The simplest example is giving your page a title that web browsers can understand, via HTML's <title> tag. Let's change the template like so:

<html>
 <head>
   <title>@@TITLE@@<title>
 <head>
 <body>
##PART##
 <body>
<html>

And change the part like so:

<--
<title>Greetings</title>
-->

Hello, world!

If this page is put through the templating system, the browser gets something like this:

<html>
 <head>
   <title>Greetings</title>
 </head>
 <body>
Hello, world!
 </body>
</html>

Notice that the title has been extracted from the part, and sucked into the template. The other tokens that can be used in substitutions are discussed in Section 3.4.

Here are some useful guidelines to keep in mind while designing a template:

  • Only use absolute URL references to style sheets and images in your templates. In other words, <style href="/path/to/my/sheet.css" /> -- see the slash on /path? This will prevent your template from breaking when you try to serve a page located in a subdirectory of the server root.

  • Use image width, height, and alt attributes particularly in _template.html to improve page loading performance.

  • Periodically validate your pages with a quality validator like those from the Web Design Group or W3C. At least your home page. You'll avoid annoying lots of people if you serve good HTML.
