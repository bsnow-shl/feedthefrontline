The Content Publishing Framework is a low-ceremony, highly-effective system for quickly developing and publishing web sites using PHP and PostgreSQL. It features a simple templating system, elegant form handling, a useful database model, and facilities for user management and role-based permissions. There's a built-in discussion forum and document repository as an application of all these subsystems. It's mostly modular and designed to be easy to extend, merge in future improvements, and support concurrent development using CVS.

The framework's driving value is effectiveness. There's no idealism here in any form: no object-oriented nirvana, no squandered effort on database abstraction, and it does not attempt to be all things to all people. It's meant for people who have to quickly develop and operate a dynamic web site, and don't mind using a system with some opinions about how things should be done.

Some of the consequences of this philosophy include a complete lack of support for MySQL, or any other database. The system has been integrated with other codebases that themselves use MySQL, but the core depends loosely but irretrievably on PostgreSQL.

Its original author is Stephen van Egmond, who wrote the system based on his experience with complex, large-scale web development, and the practical, small details that can make it easy for those involved. Support is provided through Tiny Planet Consulting, a consulting company that builds standards-based infrastructure, and advises on software development process and technique. Other developers, notably Bill Snow, have contributed to its development.

1.1. Software requirements

This system requires recent versions of Apache, PHP, and PostgreSQL to work. Apache 1.3.26, PHP 4.3, and PostgreSQL 7.4 have been seen to work well. The PHP modules mhash and pgsql are required, and the Xdebug extension is recommended.

Optional and recommended packages. The page testing infrastructure requires Ant 1.5 and a Java 1.4 development environment. The documentation can be built using the docbook utilities db2html and db2pdf. CVS 1.10 or some other source control system is recommended. The system cooperates well with the exim mail server and ecartis mailing list manager.

As of May 2003, if you are running the Debian "Woody" distribution, running the following command as root will load the appropriate packages into your system:

apt-get install postgresql apache php4-pgsql php4-mhash cvs ant docbook-dsssl alien

You will still have to go to Sun's web site to download the Java 1.4 development RPM -- not the runtime. Once downloaded, you can turn the RPM into a Debian package with alien: alien j2sdk*.rpm; dpkg -i j2sdk*.deb

Note: PostgreSQL must be configured with the transform_null_equals=true in /etc/postgresql/postgresql.conf.

Note: PHP should be configured with register_globals off. CPF will work with either setting, though.
