The Publishing Framework's templating system is designed to enforce a consistent look and feel across your entire site, or entire subsections of your site. It does this by providing a straightforward templating system that permits a separation between how the site looks overall, and how individual pages behave.

3.1. Terminology

You can mentally split up most web pages you see these days into two parts: navigation that changes infrequently, and page-specific material. In the CPF, the infrequently-changing stuff is termed the template and the page-specific material is termed the part. The template lives in a file called _template.html, and the part is whatever file under Apache's document root your web browser is asking for.

When the CPF begins serving a page, it starts looking backwards through the directory tree for a template to use. The first one it finds, it uses. Thus, if you have a directory structure like so:

htdocs/_template.html
htdocs/a.html

htdocs/foo/_template.html
htdocs/foo/b.html

htdocs/bar/c.html

... then when a browser requests a.html or c.html, they will be served up using the top-level _template.html. However, since foo/b.html has a template of its own, it will be served up using the template foo/_template.html.
