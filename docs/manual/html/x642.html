5.5. Multiple forms on a page

Many complex pages will have multiple forms. The CPF addresses this by letting you name each form with an identifier, which you use while defining, validating, and rendering your form.

For instance, if you have a web page to manage subscriptions to a mailing list. There's two forms, one to subscribe, one to unsubscribe. Unsubscribing requires a password, but subscribing doesn't.

form_set('subscribe', array(
        'email_address' => 'email,required'
));
form_set('unsubscribe', array(
        'email_address' => 'email,required',
        'password' => 'password,required'
));

if (form_valid('subscribe')) { ... }

if (form_valid('unsubscribe')) { ... }

And in your web page:

<h2>subscribe to our list</h2>

<h2>unsubscribe from our list</h2>
